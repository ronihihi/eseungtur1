# E-Sign Workflow App

A full-stack document e-signature app with user login, file upload, email sending, and real-time signing tracking.

## Features
- User registration & login
- Upload PDF or DOCX documents
- Add 7 team recipients with emails
- Sequential or simultaneous signing
- Email sent to each team with unique signing link
- Recipients draw their signature online (no account needed)
- Real-time progress tracking & reminders
- Audit trail per signature

---

## Setup on Replit

### 1. Create a new Replit project
- Go to replit.com → Create Repl → **Node.js**
- Delete the default files

### 2. Upload these files
Upload the following files maintaining the folder structure:
```
server.js
package.json
.env.example
public/
  index.html
  sign.html
```

### 3. Install dependencies
In the Replit Shell tab, run:
```bash
npm install
```

### 4. Set up environment variables
In Replit, go to **Secrets** (lock icon in sidebar) and add:

| Key | Value |
|-----|-------|
| `SESSION_SECRET` | any long random string |
| `SMTP_HOST` | smtp.gmail.com |
| `SMTP_PORT` | 587 |
| `SMTP_USER` | your-email@gmail.com |
| `SMTP_PASS` | your Gmail App Password |

**To get a Gmail App Password:**
1. Enable 2-Factor Authentication on your Google account
2. Go to https://myaccount.google.com/apppasswords
3. Create an app password for "Mail"
4. Use that 16-character password as SMTP_PASS

### 5. Run the app
```bash
node server.js
```
Or click the **Run** button in Replit.

### 6. Open the app
Replit will show you a public URL like `https://your-app.replit.app`

## How to use

1. **Register** or login with the demo account (admin@demo.com / demo123)
2. Click **Upload Document** 
3. Upload your PDF or DOCX
4. Enter the team names and email addresses
5. Customize the email message
6. Click **Send to All Teams**
7. Each team gets an email with a unique link to sign online
8. Track progress in the dashboard — send reminders to pending teams

---

## Tech Stack
- **Backend**: Node.js + Express
- **Database**: SQLite (via better-sqlite3) — no setup needed
- **Auth**: Session-based with bcrypt password hashing
- **Email**: Nodemailer (works with Gmail, SendGrid, etc.)
- **File storage**: Local disk (Replit persistent storage)
- **Frontend**: Vanilla HTML/CSS/JS — no frameworks needed
